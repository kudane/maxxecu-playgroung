# 🎯 MaxxECU Idle Control Target RPM (แปลเฉพาะระบบ Idle)

## 🧩 Target RPM Settings

| หัวข้อ | เนื้อหา |
|---|---|
| Idle RPM target table | ตารางกำหนด “รอบเป้าหมาย” ที่ระบบจะพยายามรักษาไว้ในทุกสถานการณ์ของ idle (เช่น ตามอุณหภูมิ) 0 |
| AC idle up RPM | ค่ารอบที่จะ “เพิ่มขึ้น” เมื่อมีการสั่งงานแอร์ (คอมแอร์ทำงาน) 1 |
| Clutch idle up RPM | ค่ารอบที่จะ “เพิ่มขึ้น” เมื่อมีสัญญาณคลัทช์ (เช่น เหยียบคลัทช์) 2 |

---

## 💡 อธิบายแบบใช้งานจริง (สำคัญกว่าแปล)

### 🎯 Idle RPM Target Table
- คือ “เป้าหมายหลัก” ที่ Closed Loop จะพยายามดันให้ถึง
- ใช้ในทุกกรณี เช่น:
  - เดินเบาปกติ
  - หลังถอนคันเร่ง
  - ตอนมีโหลดเพิ่ม

👉 ตัวอย่าง:
| Temp | Target RPM |
|---|---|
| 20°C | 1100 |
| 60°C | 900 |
| 90°C | 800 |

💬 แปลตรง ๆ:
- เครื่องเย็น → ต้องรอบสูง (กันดับ)
- เครื่องร้อน → ลดรอบให้นิ่ง

---

### ❄️ AC Idle Up RPM
- เมื่อแอร์ทำงาน → เครื่องโดนโหลดเพิ่ม
- ECU จะ “เพิ่ม target RPM” ชั่วคราว

👉 ตัวอย่าง:
- Target ปกติ = 800 rpm  
- AC idle up = +100 rpm  
→ กลายเป็น 900 rpm

💬 ถ้าไม่ตั้ง:
- เปิดแอร์ = รอบวูบ
- บางคัน = ดับให้ดูเลย (เจ็บแต่จริง)

---

### 🦶 Clutch Idle Up RPM
- ใช้ตอน “เหยียบคลัทช์ / เข้าเกียร์ว่าง”
- ป้องกันรอบตกตอนโหลดหายเร็ว

👉 ตัวอย่าง:
- ตอนวิ่งแล้วเหยียบคลัทช์  
- รอบจะไม่ร่วงฮวบ แต่ถูก “พยุง” ไว้

💬 เหมาะกับ:
- รถเกียร์ธรรมดา
- รถที่รอบตกไวตอนเปลี่ยนเกียร์

---

## 🔗 ความสัมพันธ์กับ Closed Loop (สำคัญมาก)

Target RPM = “เป้า”  
Closed Loop = “คนพยายามยิงให้โดนเป้า”

- ถ้า Target ตั้งมั่ว → Closed loop ก็มั่วตาม
- ถ้า Open loop ไม่ใกล้ → Closed loop ทำงานหนัก = รอบแกว่ง

👉 Logic จริง:
1. อ่าน Target RPM จาก table  
2. เทียบกับ RPM จริง  
3. เอา error ไปคำนวณ (PID)  
4. สั่ง “ลม / ไฟ” ให้เข้าใกล้เป้า  

---

## ⚠️ ข้อผิดพลาดที่เจอบ่อย (สายจูนโดนกันหมด)

- ตั้ง Target ต่ำเกิน → รถสั่น / ดับ
- ตั้ง Target สูงเกิน → กินน้ำมัน + เสียงเหมือนรีบไปไหน
- ไม่ใส่ AC idle up → เปิดแอร์แล้วรอบตก
- Open loop ไม่ตรง → Closed loop ไล่ไม่ทัน
